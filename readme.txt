【タイトル】
MOUNT
【ジャンル・分野】
音楽ゲーム
【製作・研究期間】
約7ヶ月
【進捗状況・完成度】
ゲームとして違和感なく遊べます。曲やシステム、アニメーションの追加を開発しているため完成度は90%としています。
【開発環境（使用言語・ライブラリ）】
C、C++、VisualStudio2019、DXライブラリ
【制作・研究目的】
シーン管理やゲームに重要な処理を学び、私の好きなジャンルで制作しました。
【制作・研究人数・担当】
制作は一人で行い、曲は友人から提供していただきました。
【工夫点】
ゲーム全体の雰囲気や世界観をイメージした画面を制作しました。ノーツの落下や間隔を意識し正確なノーツの位置を算出しました。
【反省点・改善点】
全体的にメモリ容量を余分に使ってしまっている部分があり、さらに工夫することにより容量を抑えることができると思いました。
現在、1曲のみの収録であるため曲を増やすことも考えています。
また、BPMが途中で変化する曲や変拍子に対応していないことも今後の課題の一つです。
【操作方法】
使用するデバイスはキーボードのみです。

----------タイトル画面----------
実行ファイルを起動すると最初のタイトル画面が表示されます。
ENTERキーでシーンが進み選曲画面に移行します。
ESCキーでアプリケーションを終了します。

----------選曲画面----------
上下キーで曲を、左右キーで難易度を選択しENTERキーで決定します。
決定するとジャケット画面に移行します。

LOCKと書かれているものは選択することができません。
4つ目の最高難易度は同じ曲の難易度のいずれかをクリアすることにより解禁され選択することができます。
解禁はSPACEキーで行います。

----------アンロック画面----------
短いアニメーションが流れ、その曲の選択した難易度が解禁されプレイができるようになります。

----------ジャケット画面----------
選曲画面で選択した曲名と難易度とジャケットを表示し、すぐにゲーム画面に移行します。

----------ゲーム画面----------
判定線付近に「START」が表示されるとゲームが始まります。
判定線、または音楽に合わせて対応するキーを入力するとスコアが加算されます。難易度によって降ってくるノーツの色が増え、白いノーツに対応するキーの上部に位置するキーが対応するようになります。

対応するキーは

ノーツが白い場合
左から1番目のレーンはVキー
左から2番目のレーンはBキー
左から3番目のレーンはNキー
左から4番目のレーンはMキー

ノーツが黄色い場合
左から1番目のレーンはFキー
左から2番目のレーンはGキー
左から3番目のレーンはHキー
左から4番目のレーンはJキー

ノーツが赤い場合
左から1番目のレーンはRキー
左から2番目のレーンはTキー
左から3番目のレーンはYキー
左から4番目のレーンはUキー

となります。

判定は3つあり、
タイミングよく押すとGREAT
少しズレるとOK
大きくズレる、または押さずにいるとBAD
になります。

判定により右のゲージが増えていき、最終的に70%溜まるとクリアとなります。

----------リザルト画面----------
ゲームの結果を表示します。
スコア、判定数を表示し、最後にランクが表示されます。

ENTERキーで選曲画面に戻ります。

----------その他----------
・現在収録されている曲は1曲のみとなっています。
・ゲーム画面でENTERキーを入力するとリザルト画面に移行し、その時点での結果が表示されます。
・タイトル画面だけではなく、ジャケット画面以外の画面でもESCキーを入力することによってアプリケーションを終了することができます。
